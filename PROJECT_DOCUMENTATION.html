RevWorkForce - Full Project Documentation

Repository: d:\revworkforce | Stack: Angular 16 + Spring Boot 2.7 + MySQL

1. End-to-End System Flow

Startup Flow

  • Frontend starts on http://localhost:4200 and serves Angular routes/components.
  • Backend starts on http://localhost:8080 and exposes /api/* REST endpoints.
  • DataLoader seeds roles/departments/designations/default users only when tables are empty.
  • Backend connects to MySQL database revworkforcep2.

Request Lifecycle

  • Angular component calls ApiService.
  • TokenInterceptor attaches JWT for secured APIs.
  • RequestLoggingFilter creates request id and logs HTTP_IN/HTTP_OUT.
  • JwtAuthenticationFilter validates token and sets security context.
  • Controller receives request and delegates to Service.
  • Service applies business rules, transactions, authorization checks, and calls repositories.
  • Repositories (JPA) read/write MySQL.
  • GlobalExceptionHandler converts exceptions to API responses.
  • ApplicationLoggingAspect logs controller/service/repository entry/exit/failure.

2. Business Module Flows

Authentication + Security

  • User logs in via /api/auth/login.
  • AuthService authenticates through Spring Security and creates JWT.
  • Password reset via /api/auth/reset-password validates employee + email + password pattern and stores BCrypt hash.

Leave Management

  • Employee applies leave (/api/leaves) with date checks and leave-balance checks.
  • Leave summary (/api/leaves/summary) returns total/remaining/pending metrics.
  • Manager/Admin approves or rejects pending requests only.
  • Related notifications are generated for manager/employee.

Performance + Goals

  • Manager can submit review for direct reportees; Admin can submit for employees/managers.
  • Employee view is read-only with rating visualization and details panel in UI.
  • Goal status changes allowed for owner/manager, and COMPLETED cannot be reverted.

Notifications + Admin

  • Admin can send broadcast or specific notifications; manager/employee can send to one user.
  • Read/unread state updated by recipient.
  • Admin maintains departments, designations, holidays, announcements, user activation, and manager assignment.

4. File-by-File Reference (Current Core Code)

Backend Main Files

  • RevWorkforceApplication.java - application entry point.
  • config/DataLoader.java - bootstrap seed data.
  • config/SecurityConfig.java - security filter chain, CORS, auth manager, encoder.
  • controller/* - API endpoints per module (auth/users/leaves/goals/performance/admin).
  • dto/* - request/response contracts and validation constraints.
  • entity/* - JPA mapping to MySQL tables.
  • enums/* - domain constants (roles/status/priorities/types).
  • exception/* - centralized exception handling and custom exceptions.
  • logging/ApplicationLoggingAspect.java - AOP method logs.
  • logging/RequestLoggingFilter.java - request correlation + HTTP logs.
  • repository/* - Spring Data JPA repositories.
  • security/AppUserDetailsService.java - user details bridge for Spring Security.
  • security/JwtAuthenticationFilter.java - token parsing/validation per request.
  • security/JwtUtil.java - token generation/validation helper.
  • service/* - business logic and transaction boundaries.

Backend Resource Files

  • backend/src/main/resources/application.yml - MySQL, JPA, server, JWT settings.
  • backend/src/main/resources/log4j2.xml - console + rolling-file logging config.
  • backend/pom.xml - dependencies/build plugins.

Frontend Main Files

  • src/main.ts - Angular bootstrap.
  • src/styles.css - global theme and UI animations.
  • app/app.module.ts - root Angular module declarations/imports.
  • app/app-routing.module.ts - route definitions and guards.
  • app/app.component.ts - shell layout/nav/route outlet.
  • app/components/landing.component.ts - public landing + login/reset panel.
  • app/components/dashboard.component.ts - role-aware dashboard + notifications/profile.
  • app/components/leave.component.ts - leave apply/list/summary UI.
  • app/components/performance.component.ts - review create/view with detail panel.
  • app/components/goals.component.ts - goals creation and status tracking.
  • app/components/admin.component.ts - admin management screens.
  • app/components/directory.component.ts - team directory/search.
  • app/services/api.service.ts - HTTP wrapper.
  • app/services/auth.service.ts - login/logout/token persistence.
  • app/services/token.interceptor.ts - JWT header injection.
  • app/services/ui-state.service.ts - UI state coordination.
  • app/guards/auth.guard.ts - route auth guard.
  • app/guards/role.guard.ts - role-based route guard.
  • src/environments/environment.ts - API base URL config.
  • src/assets/* - logo/hero assets.

Backend Test Files

  • controller/*Test.java - controller unit tests via standalone MockMvc.
  • service/*Test.java - service unit tests using JUnit4 + Mockito.

5. Developer-wise Segregated Flow (Final Team Structure)

DEV 1 - Authentication & Security Lead

  • Owns: login/reset-password, JWT, BCrypt, authorization rules, security hardening.
  • Primary files: AuthController, AuthService, SecurityConfig, Jwt*, guards/interceptor.
  • Flow: User credentials -> Spring auth -> JWT -> secured API access.
  • Branch: feature/dev1-auth-security

DEV 2 - Employee Experience Lead

  • Owns: dashboard UX, profile edit, directory/search UI, responsive behavior.
  • Primary files: dashboard.component.ts, directory.component.ts, styles.css (coordination).
  • Flow: Login -> dashboard metrics/profile -> team directory/search.
  • Branch: feature/dev2-employee-experience

DEV 3 - Leave Management Workflow Lead

  • Owns: leave apply/cancel/approve/reject, leave balance/summary, transaction consistency.
  • Primary files: LeaveController, LeaveService, LeaveRequest*, leave.component.ts.
  • Flow: Employee apply -> manager/admin decision -> status + notification.
  • Branch: feature/dev3-leave-workflow

DEV 4 - Performance & Goals Lead

  • Owns: performance review lifecycle, goals and feedback validation.
  • Primary files: Performance*, Goal*, performance.component.ts, goals.component.ts.
  • Flow: Manager/Admin submits review -> employee views details; goals tracked to completion.
  • Branch: feature/dev4-performance-goals

DEV 5 - System Owner

  • Owns: admin management, announcements/notifications, schema/docs, logging/quality gates.
  • Primary files: Admin*, User*, Notification*, docs files, logging files.
  • Flow: Master data maintenance -> organization-wide governance -> documentation + release readiness.
  • Branch: feature/dev5-system-owner

6. Merge Plan to Avoid Errors

  • Merge order: Dev1 -> Dev3 -> Dev4 -> Dev2 -> Dev5.
  • Before each PR: run backend tests + frontend build.
  • Rebase each feature branch on latest main before merge.
  • Use focused commits and avoid cross-ownership file edits.
  • Final gate: smoke test login, leave, notifications, performance, goals, admin CRUD.

7. Commands (Reference)

  • Backend run: cd backend && mvn spring-boot:run
  • Frontend run: cd frontend && npm install && npm start
  • Backend tests: cd backend && mvn test (or mvnd -q test if configured)
